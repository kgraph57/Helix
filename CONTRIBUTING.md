# 一緒にHelixを育てませんか？（貢献ガイド）

Helixに興味を持っていただき、ありがとうございます！

このプロジェクトは、皆さんのような現場の医療従事者からのフィードバックやアイデアによって、もっともっと便利になります。「こんなプロンプトがあったら助かる！」「ここ、もっとこうしたら分かりやすいかも」そんな小さな気づきが、Helixを大きく成長させます。

このドキュメントでは、開発者ではない先生方でも、気軽にプロジェクトに参加できる方法をご紹介します。

## どんな貢献ができるの？

プログラミングの知識は、まったく必要ありません。先生の専門知識や、日々の臨床での経験そのものが、最高の貢献になります。

### 💡 アイデア提案

**「こんなプロンプト、どう？」**
- 「〇〇な患者さんの説明用に、こんなプロンプトがあったら便利なのに」
- 「このプロンプト、もっとこう質問したら精度が上がったよ」

**「こんな機能があったら最高！」**
- 「キーワードでプロンプトを検索できる機能がほしい」
- 「よく使うプロンプトを、お気に入り登録したい」

**「ここ、ちょっと分かりにくいかも…」**
- 「このガイドの説明、専門的すぎて少し難しいです」
- 「ここの日本語、ちょっと不自然な感じがします」

どんな些細なことでも構いません。先生が感じたことを、ぜひ教えてください。

### 💬 提案の方法

提案は、GitHubの「**Issues**」という機能を使います。掲示板に書き込むようなイメージです。

1. [HelixのIssuesページ](https://github.com/kgraph57/Helix/issues)にアクセスします。
2. 「**New issue**」ボタンをクリックします。
3. 「**Feature request**」や「**Bug report**」など、目的に合ったテンプレートを選びます。
4. タイトルと内容を、分かる範囲で記入して「**Submit new issue**」をクリックすれば完了です！

## 開発者の方へ (For Developers)

もちろん、コードでの貢献も大歓迎です！

### 開発環境のセットアップ

```bash
# 1. リポジトリを自分のPCにコピーします
git clone https://github.com/kgraph57/Helix.git
cd Helix

# 2. 必要なツールをインストールします
pnpm install

# 3. 開発用のサーバーを起動します
pnpm dev
```

あとは、ブラウザで `http://localhost:5173` を開けば準備完了です。

### 開発の進め方

1. **ブランチを作る**: `main`ブランチから、作業用のブランチを切ります。ブランチ名は、`feature/〇〇` や `fix/〇〇` のように、何をするか分かりやすく付けましょう。
   ```bash
   git checkout -b feature/your-amazing-idea
   ```

2. **コミットする**: 変更内容が分かるように、こまめにコミット（記録）します。コミットメッセージは、[Conventional Commits](https://www.conventionalcommits.org/)の形式に従うと、他の人が見たときに分かりやすいです。
   - `feat:` 新しい機能
   - `fix:` バグ修正
   - `docs:` ドキュメントの変更

3. **プルリクエストを作る**: 変更が終わったら、`main`ブランチへのマージをリクエスト（プルリクエスト）します。変更内容の要約や、関連するIssue番号を記載してください。

### プルリクエスト前のチェックリスト

- [ ] `pnpm test` で、テストがすべて通ること
- [ ] `pnpm check` で、型エラーが出ないこと
- [ ] `pnpm format` で、コードがきれいに整形されていること

## 🙏 最後に

あなたの少しのアクションが、Helixを、そして未来の医療をより良くする大きな一歩になります。貢献に心から感謝します！
