# コントリビューションガイド

Medical Prompt Hubへのコントリビューションに興味を持っていただき、ありがとうございます。このガイドでは、プロジェクトへの貢献方法を説明します。

## コントリビューションの方法

Medical Prompt Hubへの貢献には、様々な方法があります。

### 新しいプロンプトの追加

医療現場で役立つ新しいプロンプトを作成し、リポジトリに追加できます。診断、治療、コミュニケーション、文献レビュー、症例分析など、様々なカテゴリのプロンプトを歓迎します。

### 既存プロンプトの改善

既存のプロンプトをより効果的に、より使いやすくするための改善提案も歓迎します。実際の使用経験に基づいたフィードバックは特に貴重です。

### ドキュメントの充実

使用例の追加、ベストプラクティスの共有、FAQの更新など、ドキュメントの改善も重要なコントリビューションです。

### バグ報告と機能提案

プロンプトの問題点や、新しい機能のアイデアがあれば、GitHubのIssuesで報告してください。

## プロンプト作成のガイドライン

新しいプロンプトを作成する際は、以下のガイドラインに従ってください。

### 構造

各プロンプトファイルは、以下の構造に従って作成してください。

まず**概要**セクションで、プロンプトの目的と用途を簡潔に説明します。次に**プロンプトテンプレート**セクションで、実際に使用するプロンプトの文章を記載します。入力が必要な部分は`[項目名]`の形式で示してください。

**使用例**セクションでは、具体的な入力例を提示します。実際の臨床場面を想定した現実的な例を示してください（ただし、実在の患者情報は使用しないでください）。

**ベストプラクティス**セクションでは、効果的な使い方のコツや注意点を記載します。**カスタマイズ**セクションでは、異なる状況への応用方法を説明します。最後に**関連プロンプト**セクションで、関連する他のプロンプトへのリンクを提供します。

### 品質基準

プロンプトは以下の基準を満たす必要があります。

**医学的正確性**は最も重要です。プロンプトの内容は、現在の医学的知識とエビデンスに基づいている必要があります。可能であれば、参考文献やガイドラインを明記してください。

**明確性**も重要です。プロンプトの指示は明確で、曖昧さがないようにしてください。医療従事者が容易に理解し、使用できる内容にします。

**実用性**を重視してください。実際の医療現場で使用できる、実践的な内容にしてください。理論的すぎる内容や、実現不可能な内容は避けます。

**安全性**への配慮も必須です。プロンプトには必ず適切な免責事項と注意事項を含めてください。AIの出力が医療判断を代替するものではないことを明記します。

**包括性**も考慮してください。可能な限り、様々な患者背景や臨床状況に対応できる柔軟なプロンプトを作成してください。

### ファイル命名規則

プロンプトファイルは、内容を表す簡潔な英語名を使用し、小文字とハイフンで区切ります。例: `differential-diagnosis.md`、`patient-education.md`

適切なカテゴリディレクトリに配置してください。新しいカテゴリが必要な場合は、Issueで提案してください。

## コントリビューションのプロセス

### 1. Issueの作成

大きな変更を行う前に、まずGitHub Issueを作成して、提案内容を説明してください。これにより、重複作業を避け、コミュニティからのフィードバックを得ることができます。

### 2. フォークとブランチ作成

リポジトリをフォークし、作業用のブランチを作成します。ブランチ名は、作業内容を表す簡潔な名前にしてください（例: `add-pediatric-diagnosis-prompt`、`improve-patient-education`）。

### 3. 変更の実装

ガイドラインに従って、プロンプトやドキュメントを作成・編集します。コミットメッセージは、変更内容を明確に説明するものにしてください。

### 4. プルリクエストの作成

変更が完了したら、プルリクエスト（PR）を作成します。PRの説明には、以下の情報を含めてください。

変更の目的と内容を説明し、関連するIssue番号があれば記載します。新しいプロンプトの場合は、どのような臨床場面で使用することを想定しているかを説明してください。テストした内容があれば、その結果を共有します。

### 5. レビューと修正

メンテナーやコミュニティメンバーがPRをレビューします。フィードバックがあれば、必要に応じて修正を行ってください。建設的な議論を歓迎します。

### 6. マージ

レビューが承認されたら、PRがマージされます。あなたのコントリビューションがプロジェクトの一部になります！

## 行動規範

Medical Prompt Hubのコミュニティは、すべての参加者にとって安全で歓迎される場所であるべきです。

**尊重**を持って接してください。異なる意見や経験を尊重し、建設的な議論を心がけます。

**専門性**を維持してください。医療に関する情報を扱うため、専門的で責任ある態度を保ちます。

**協力的**であってください。他のコントリビューターを支援し、知識を共有します。

**プライバシー**を保護してください。実際の患者情報や個人を特定できる情報は決して共有しないでください。

## 質問やサポート

コントリビューションに関して質問がある場合は、以下の方法でお問い合わせください。

GitHub Issuesで質問を投稿するか、既存のIssueにコメントしてください。メンテナーやコミュニティメンバーがサポートします。
