## 📖 Helixって、なに？

Helixは、医師や研究者の皆さんが、ChatGPTやClaudeといったAIを日々の臨床や研究、教育の現場で最大限に活用するための、いわば**「AIへの指示書の教科書」**です。

「Helix（らせん）」という名前は、生命の設計図であるDNAの二重らせん構造から名付けました。DNAが生命の基本情報を持つように、Helixは、先生方がより質の高い医療を提供するための「知恵」と「サポート」を提供します。

私たちの哲学はとてもシンプルです。

> **AIは、医師に取って代わるものではない。医師の能力を拡張するものだ。**

先生方が持つ、臨床での判断力、経験、そして患者さんへの共感。これらはAIには決して真似できません。Helixは、先生方のそんな「人間らしい強み」を最大限に活かすために、日々の雑務や定型業務をスマートに肩代わりします。

### Helixで、できること

Helixは、一言で言えば**「AIに的確な質問をするための、実践的なカンペ集」**です。

- **100以上の専門プロンプト**: 診断で悩んだ時、治療計画を立てる時、論文を書きたい時…。医療現場の「あるある」な場面で、すぐに使える指示書が揃っています。
- **カテゴリ別整理**: 「診断支援」「書類作成」など10のカテゴリに分かれているので、まるでカルテ棚から必要な情報を探すように、目的のプロンプトがすぐに見つかります。
- **コピペでOK**: 面倒な入力は不要です。プロンプトをコピーして、いつもお使いのAIチャットに貼り付けるだけ。
- **実践的なガイド**:「症例報告ってどう書くの？」「統計解析ってどうやるの？」そんな悩みを解決する、具体的な手順書も用意しました。
- **41のAI活用テクニック**: AIの実力を120%引き出すための「プロンプトエンジニアリング」のコツを、医療者向けにわかりやすく解説します。

## ✨ たとえば、こんな使い方

### 📚 プロンプトライブラリ

100以上のプロンプトが、あなたの「知りたい」「やりたい」をサポートします。

- **診断に迷ったとき**: 「この症状、他にどんな病気が考えられる？」とAIに壁打ちできます。
- **治療計画を立てるとき**: 最新のエビデンスに基づいた治療の選択肢を、AIがリストアップしてくれます。
- **面倒な書類作成**: 紹介状や退院サマリー、インシデントレポートも、AIと一緒ならサクッと片付きます。
- **患者さんへの説明**: 少し難しい病状の話も、AIを使えば、患者さんに伝わるやさしい言葉に変換できます。
- **論文リサーチ**: 大量の論文も、AIが要点をまとめてくれるので、時短になります。

### 🎯 ワークフローガイド

一人では大変な作業も、Helixのガイドがあれば、まるで先輩医師が隣で教えてくれるように、ステップバイステップで進められます。

- **症例報告ワークフロー**: 「初めてで何から手をつけていいかわからない…」という先生も、このガイド通りに進めれば、論文投稿までたどり着けます。
- **統計解析サポート**: 苦手な人が多い統計も、AIにコードを書いてもらえば怖くありません。
- **学会発表の準備**: 抄録作りから、当日の質疑応答対策まで、まるっとサポートします。

## 🚀 さあ、はじめましょう！

使い方は、とっても簡単です。

1. **[Live Demo](https://kgraph57.github.io/Helix/)** にアクセスします。
2. あなたの目的に合ったプロンプトを探します。
3. プロンプトをコピーして、ChatGPTなどのAIチャットに貼り付けます。
4. プロンプト内の`[ ]`の部分を、あなたのケースに合わせて書き換えるだけ！

```
【使用例】

「診断支援」カテゴリから「鑑別診断ジェネレーター」を選ぶ
↓
プロンプトをコピー
↓
ChatGPTに貼り付け
↓
患者さんの情報を入力
↓
あっという間に、鑑別診断のリストが完成！
```

## 🛠️ For Developers

### Tech Stack

- **Frontend**: React 19, TypeScript, Tailwind CSS, shadcn/ui, Radix UI, Framer Motion
- **Build & Development**: Vite 7, Wouter, Vitest, React Testing Library
- **Deployment**: GitHub Pages, GitHub Actions

### Development

**Prerequisites**
- Node.js 20+
- pnpm 10+

**Installation**
```bash
# 1. リポジトリをクローン
git clone https://github.com/kgraph57/Helix.git
cd Helix

# 2. 依存関係をインストール
pnpm install

# 3. 開発サーバーを起動
pnpm dev

# 4. ブラウザで http://localhost:5173 を開く
```
