# ステップ2: 修正の具体的ルールを学ぶ

## 📋 このステップの目的

トーンマナー修正の核心部分です。このステップでは、実際にどのように文章を修正するのか、7つの具体的なルールを学びます。

## ⏱️ 所要時間

20分

## 🎯 達成基準

- 7つの修正ルールを理解し、それぞれの例を見て違いがわかる
- 実際のドキュメントで、どのルールを適用すべきか判断できる

## 📝 詳細手順

### ルール1: 太字（**bold**）の削除

**理由**: 太字の多用は圧迫感を与え、読みづらくなります。本当に重要な箇所のみに限定しましょう。

**修正前**:
```markdown
**重要なポイント**: この症例は...
**注意**: 必ず確認してください
**手順1**: データを収集する
```

**修正後**:
```markdown
重要なポイント: この症例は...
注意: 必ず確認してください
手順1: データを収集する
```

**適用タイミング**: ドキュメント全体を通して、太字を見つけたら削除します。ただし、見出しやタイトルは除きます。

### ルール2: 語尾の柔らかさ

**理由**: 「です・ます」調で統一することで、親しみやすさを出します。

**修正前**:
```markdown
以下の手順を実行する。
データを収集する必要がある。
論文を読む。
```

**修正後**:
```markdown
以下の手順を実行します。
データを収集する必要があります。
論文を読みます。
```

**適用タイミング**: 文末が「〜する」「〜である」になっている箇所を見つけたら、「〜します」「〜です」に変更します。

### ルール3: 具体例の追加

**理由**: 抽象的な表現を避け、具体的なイメージを持たせることで、理解を助けます。

**修正前**:
```markdown
稀な疾患や病態を選ぶ。
```

**修正後**:
```markdown
稀な疾患や病態を選びます。教科書でしか見たことのないような、珍しい疾患や病態です。
```

**適用タイミング**: 抽象的な表現を見つけたら、具体例を追加します。

### ルール4: 比喩・たとえ話の活用

**理由**: 比喩を使うことで、読者の理解を助け、記憶に残りやすくします。

**修正前**:
```markdown
診断に苦労した症例
```

**修正後**:
```markdown
診断に苦労した症例。まるで推理小説のように、診断までの道のりが曲がりくねっていた症例です。
```

**適用タイミング**: 複雑な概念や、イメージしにくい内容を説明するときに使います。

### ルール5: 読者への語りかけ

**理由**: 「〜してください」「〜してみましょう」など、読者に直接語りかける表現を使うことで、親しみやすさを出します。

**修正前**:
```markdown
以下の基準を参考にする。
論文を読む。
```

**修正後**:
```markdown
以下の基準を参考にしてみてください。
論文を読んでみましょう。
```

**適用タイミング**: 読者に行動を促す箇所で使います。

### ルール6: 感情表現の追加

**理由**: 感情を込めることで、読者の共感を得やすくします。

**修正前**:
```markdown
重要な学びが含まれている症例
```

**修正後**:
```markdown
「これは、みんなに知ってほしい！」と思った症例
```

**適用タイミング**: 読者の共感を得たい箇所で使います。

### ルール7: 箇条書きの工夫

**理由**: 単なるリストではなく、説明を加えることで、理解を深めます。

**修正前**:
```markdown
- 新規性
- 意外性
- 教育的価値
```

**修正後**:
```markdown
1. 新規性（Novelty）: 未知の疾患、新しい原因、新しい治療法などが含まれているか？
2. 意外性（Unexpectedness）: 予期せぬ関連や、予期せぬ出来事があるか？
3. 教育的価値（Educational Value）: 診断の落とし穴や、治療のヒントなど、読者が明日からの診療に活かせる学びがあるか？
```

**適用タイミング**: 箇条書きを見つけたら、説明を追加します。

## 🎯 実践例

実際のドキュメントで、これらのルールを適用してみましょう。

### 修正前

```markdown
### 📋 このステップの目的

**症例の選定**は症例報告の第一歩である。報告する価値のある症例を見つけることが重要だ。

### 📝 詳細手順

#### 手順1: 候補症例をリストアップ

以下の観点で候補をリストアップする：
- **診断に苦労した症例**
- **治療に難渋した症例**
- **稀な疾患や病態**
```

### 修正後

```markdown
### 📋 このステップの目的

症例報告の第一歩は、数ある経験の中から「報告する価値のある」一例を見つけ出すことです。すべての症例が報告に適しているわけではありません。このステップでは、どんな症例が国際的な学術コミュニティにとって価値があるのかを判断する基準を学びます。

### 📝 詳細手順

#### 手順1: 候補症例をリストアップする

まずは、最近1〜2年で経験した症例の中から、いくつかの観点で候補をリストアップします。

こんな症例が候補になります：

- 診断に苦労した症例: 鑑別診断が複雑だったり、診断の決め手に意外な発見があった症例。まるで推理小説のように、診断までの道のりが曲がりくねっていた症例です。
- 治療に難渋した症例: 標準治療が効かなかった、予期せぬ副作用が出た、あるいは新しい治療法を試したなど。教科書通りにいかなかった症例です。
- 稀な疾患や病態: 教科書でしか見たことのないような、珍しい疾患や病態。
```

### 適用したルール

- **ルール1**: 太字を削除（「**症例の選定**」→「症例の選定」）
- **ルール2**: 語尾を柔らかく（「〜である」→「〜です」）
- **ルール3**: 具体例を追加（「すべての症例が報告に適しているわけではありません」）
- **ルール4**: 比喩を追加（「まるで推理小説のように」）
- **ルール5**: 読者への語りかけ（「リストアップします」）
- **ルール7**: 箇条書きに説明を追加

## 💡 ヒント

- **一度に全部やらない**: 最初は1つのルールに集中して、慣れてきたら他のルールも適用していきましょう。
- **before/afterを比較する**: 修正前と修正後を並べて比較すると、違いがよくわかります。
- **声に出して読む**: 修正後の文章を声に出して読んでみてください。読みやすければ、良い修正です。
