# はじめに：読みやすく、親しみやすいドキュメントを作ろう

## このガイドで得られること

「ドキュメント、もっと読みやすくしたいな…でも、どう書けばいいんだろう？」

もし先生がそう思っているなら、このガイドはきっと役に立ちます。

このガイドは、Helixの全ドキュメントを「Kensuu-style」の親しみやすいトーンに統一するための指示書です。医療従事者が気軽に読めて、実践的に使える文書を目指します。

## そもそも「トーンマナー」って、なんだっけ？

簡単に言えば、「文章の雰囲気や語り口」のことです。

同じ内容でも、書き方次第で、読者の印象はガラッと変わります。

- 堅苦しい文章: 「以下の手順を実行する。データを収集する必要がある。」
- 親しみやすい文章: 「以下の手順を実行します。データを収集する必要があります。」

たったこれだけの違いですが、後者の方が読みやすく、親しみやすいですよね。

| 項目 | 一言でいうと… |
|---|---|
| 目的 | 読者が気軽に読めて、すぐに実践できる文書を作る |
| 対象 | Helixの全ドキュメント |
| 特徴 | Kensuu-style、Notion/YouMind、Manusを参考にした親しみやすいトーン |
| なぜ重要？ | 読みやすい文書は、読者の理解を助け、実践につながるから |

## なんで、トーンマナーを統一するの？

「今のままでも十分じゃない？」と思いますよね。でも、トーンマナーを統一することには、たくさんのメリットがあるんです。

- **読みやすさ**: 統一されたトーンは、読者の負担を減らし、理解を助けます。
- **親しみやすさ**: 堅苦しくない文章は、読者の心理的なハードルを下げます。
- **実践的**: 具体例や比喩を使うことで、読者がすぐに実践できるようになります。
- **ブランディング**: 統一されたトーンは、Helixのブランドイメージを強化します。

そして何より、読者が「このドキュメント、わかりやすい！」と感じてくれる。これって、すごく嬉しいことですよね？

## 全体像はこんな感じ

このガイドでは、トーンマナー修正のプロセスを以下のステップに分けました。

| ステップ | やること | 所要時間（目安） |
|---|---|---|
| 1 | 目標とする文体を理解する | 10分 |
| 2 | 修正の具体的ルールを学ぶ | 20分 |
| 3 | 避けるべき表現を確認する | 10分 |
| 4 | 修正の手順を実践する | 30分〜 |
| 5 | 図解を挿入する | 20分 |

## 参考スタイル

このガイドでは、以下のスタイルを参考にしています。

- **Kensuu (梅田健太郎)**: nanapi創業者、起業家。わかりやすく実践的な文章で知られる
- **Notion/YouMind**: 親しみやすく、視覚的にわかりやすいドキュメント
- **Manus**: ユーザーフレンドリーで実用的なガイド

## このガイドの使い方

1. **まずは全体を読む**: ざっと目を通して、トーンマナーの全体像を把握してください。
2. **ルールを確認する**: 修正の具体的ルールを確認して、どんな修正が必要か理解してください。
3. **実践する**: 実際にドキュメントを修正してみてください。
4. **フィードバックを得る**: 修正したドキュメントを他の人に読んでもらい、フィードバックをもらってください。

さあ、一緒に読みやすいドキュメントを作っていきましょう！
