# ステップ1: 臨床の疑問をPICOで構造化する

## EBMの第一歩は「問い」を立てること

EBM（Evidence-Based Medicine）の実践は、日々の臨床現場で生まれる「疑問」から始まります。
この漠然とした疑問を、検索可能な形に整理するためのフレームワークがPICOです。

適切な「問い」を立てることが、良質なエビデンスにたどり着くための最も重要なステップです。

---

## PICOとは？

PICOは、臨床上の疑問を4つの要素に分解し、構造化するためのツールです。

PはPatientまたはProblem（患者・問題）を表します。
どのような患者が対象かを示します。
例として、2型糖尿病の成人患者が挙げられます。
IはIntervention（介入）を表します。
検討したい治療法や検査を示します。
例として、SGLT2阻害薬の投与が挙げられます。
CはComparison（比較）を表します。
何と比較するかを示します。
プラセボ、標準治療などが比較対象となります。
例として、プラセボまたは他の血糖降下薬が挙げられます。
OはOutcome（結果）を表します。
評価したい結果を示します。
例として、主要心血管イベント（心筋梗塞、脳卒中など）の発生率が挙げられます。

このように疑問を構造化することで、検索キーワードが明確になり、論文検索の精度が格段に向上します。

---

## 様々なタイプの臨床疑問とPICO

PICOは、治療に関する疑問だけでなく、診断、予後、原因など、様々なタイプの臨床疑問に応用できます。

### 1. 治療 (Therapy)

疑問として、「高齢の高血圧患者に、サイアザイド系利尿薬とCa拮抗薬、どちらが脳卒中予防に有効か？」が挙げられます。
Pは高齢の高血圧患者です。
IはCa拮抗薬です。
Cはサイアザイド系利尿薬です。
Oは脳卒中の発生率です。

### 2. 診断 (Diagnosis)

疑問として、「深部静脈血栓症（DVT）が疑われる患者に、D-dimer測定は診断の一助となるか？」が挙げられます。
PはDVTが疑われる患者です。
IはD-dimer測定です。
CはD-dimer測定なし、または超音波検査のみです。
OはDVTの診断精度（感度・特異度）です。

### 3. 予後 (Prognosis)

疑問として、「急性心筋梗塞後、心不全を合併した患者の5年生存率は？」が挙げられます。
Pは急性心筋梗塞後、心不全を合併した患者です。
Iは特定の介入なしです。
Cは特定の比較なしです。
Oは5年生存率です。

### 4. 原因 (Etiology / Harm)

疑問として、「経口避妊薬の服用は、静脈血栓塞栓症のリスクを増加させるか？」が挙げられます。
Pは若年女性です。
Iは経口避妊薬の服用です。
Cは経口避妊薬の非服用です。
Oは静脈血栓塞栓症の発生率です。

## AIを使って臨床疑問をPICOに変換する

日常の臨床で浮かんだ漠然とした疑問を、AIに入力するだけで、構造化されたPICO形式に変換させることができます。

### 📋 PICO変換プロンプト（コピペ用）

```
あなたはEBMに精通した臨床医です。以下の日常的な臨床疑問を、PICO形式に構造化してください。

【臨床疑問】
[ここに、普段感じている疑問を自然な文章で入力]

【出力形式】
以下の形式で出力してください。

* P (Patient/Problem):
* I (Intervention):
* C (Comparison):
* O (Outcome):
```

### 実践例

入力プロンプト:
```
あなたはEBMに精通した臨床医です。以下の日常的な臨床疑問を、PICO形式に構造化してください。

【臨床疑問】
最近、外来に来た心房細動の患者さんに、新しい抗凝固薬（DOAC）と昔からのワルファリン、どっちを使うか迷っています。どっちが脳梗塞をよく防げて、安全なんですかね？
```

AIの出力例:
```
はい、承知いたしました。その臨床疑問をPICO形式に構造化します。

* P (Patient/Problem): 非弁膜症性心房細動の患者
* I (Intervention): 直接経口抗凝固薬（DOAC）の投与
* C (Comparison): ワルファリンの投与
* O (Outcome): 脳卒中または全身性塞栓症の発生率（有効性）、および重大な出血イベントの発生率（安全性）
```

このように、普段の言葉で問いかけるだけで、AIが的確なPICOを作成してくれます。
これにより、次のステップである論文検索の準備が整います。
